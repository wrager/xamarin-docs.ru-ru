# <a name="alternate-resources"></a>Альтернативный ресурсы

Альтернативный ресурсы — это ресурсы, предназначенные определенному устройству или конфигурацию времени выполнения, например текущий язык, определенном экране размер или зависимости от плотности пикселей. Если Android можно сопоставить ресурс, который больше подходит для конкретного устройства или конфигурации ресурса по умолчанию, то этот ресурс будет использоваться вместо этого. Если не удается найти альтернативных ресурсов, который соответствует текущей конфигурации, ресурсов по умолчанию будут загружены. Как Android решает, какие ресурсы будут использоваться приложением рассматриваются более подробно ниже в разделе расположение ресурса

Альтернативный ресурсы организованы в виде подкаталога в папке ресурсов с точки зрения типа ресурса, так же, как ресурсы по умолчанию. Имя подкаталога альтернативный ресурс находится в форме: _ResourceType_-_квалификатор_

*Квалификатор* — имя, которое определяет конфигурацию конкретного устройства.
Может существовать более одного квалификатора в имени каждого из них и разделены дефисами. На следующем снимке экрана показано простого проекта, имеющего альтернативный ресурсы для различных конфигураций, таких как языкового стандарта, плотность экрана, размер экрана и ориентации:

При добавлении квалификаторы типа ресурса, применяются следующие правила:

1. Может существовать более одного квалификатора, с каждой квалификатор, разделенных дефисом.

2. Квалификаторы, может быть указан только один раз.

3. Квалификаторы должен быть в том порядке, в котором они указаны в следующей таблице.

Ниже перечислены возможные квалификаторов для ссылки.

- **MCC и кодом МНЧ** &ndash; [код страны мобильных](http://en.wikipedia.org/wiki/List_of_mobile_country_codes) (MCC) и при необходимости [кода мобильной сети](http://en.wikipedia.org/wiki/Mobile_Network_Code) (кодом МНЧ). Сеть, к которой устройство подключено к предоставит кодом МНЧ будет предоставляют MCC SIM-карта. Несмотря на то, что существует возможность целевых языков, использующих код страны для мобильных устройств, рекомендуется подход является использование квалификатора языка, указанными ниже. Например, чтобы целевые ресурсы в Германии, будет квалификатор `mcc262`. Чтобы целевые ресурсы для T-Mobile в США квалификатор не `mcc310-mnc026`.
  Полный список кодов мобильной сети и коды стран мобильных устройств в разделе <http://mcclist.com/>.

- **Язык** &ndash; двухбуквенный [код ISO 639-1 языка](http://en.wikipedia.org/wiki/ISO_639-1) и при необходимости сопровождаемый двухбуквенный [код региона ISO 3166-альфа 2](http://en.wikipedia.org/wiki/ISO_3166-1_alpha-2). 
  Если оба квалификаторы предоставляются, то они разделяются `-r`. Например, чтобы целевой французский язык язык и региональные стандарты, а затем квалификатор `fr` используется. Целевой язык и региональные стандарты французский — канадский, `fr-rCA` будет использоваться. Полный список кодов языка и региона см. в разделе [коды для представления из названий языков](http://www.loc.gov/standards/iso639-2/php/English_list.php) и [названия стран и элементы кода](http://www.iso.org/iso/country_codes/iso_3166_code_lists/country_names_and_code_elements.htm).

- **Наименьшее значение толщины** &ndash; задает наименьшую ширину экрана приложение предназначено для выполнения. Более подробно рассматривается в [Создание ресурсов для экранов с разными](~/android/app-fundamentals/resources-in-android/resources-for-varying-screens.md). 
  Этот параметр доступен на уровне API 13 (Android 3.2) и более поздних версий. Например, описатель `sw320dp` используется для целевых устройств, высота и ширина составляет не менее 320dp.

- **Доступную ширину** &ndash; Минимальная ширина экрана в формате w*N*точки распространения, где *N* имеет ширину в плотность независимых пикселях.
  Это значение может изменяться при повороте устройства. Более подробно рассматривается в [Создание ресурсов для экранов с разными](~/android/app-fundamentals/resources-in-android/resources-for-varying-screens.md). 
  Доступность на уровне API 13 (Android 3.2) и более поздних версий. Пример: квалификатор w720dp используется для целевых устройств, которые имеют ширину наименьших 720dp.

- **Доступная высота** &ndash; минимальную высоту экрана в формате h*N*точки распространения, где *N* является высота в точки распространения. Это значение может изменяться при повороте устройства. Более подробно рассматривается в [Создание ресурсов для экранов с разными](~/android/app-fundamentals/resources-in-android/resources-for-varying-screens.md). 
  Доступность на уровне API 13 (Android 3.2) и более поздних версий. Например h720dp квалификатор используется для целевых устройств, которые имеют высоту наименьших 720dp

- **Размер экрана** &ndash; этот квалификатор представляет собой обобщение размера экрана, которые эти ресурсы для. Он является более подробно рассматривается в [Создание ресурсов для экранов с разными](~/android/app-fundamentals/resources-in-android/resources-for-varying-screens.md). 
  Допустимые значения: `small`, `normal`, `large` и `xlarge`. Добавлено в API уровня 9 (Android 2.3/Android 2.3.1/Android 2.3.2)

- **Экран аспект** &ndash; основывается на пропорции не ориентацию экрана. Длинные экрана шире. Добавить API уровня 4 (Android 1.6). Возможными значениями являются долго и notlong.

- **Ориентация экрана** &ndash; книжной и альбомной ориентации экрана. Это можно изменить во время существования приложения.
  Возможными значениями являются `port` и `land`.

- **Закрепить режим** &ndash; для устройств в автомобиле закрепления или закрепить службы поддержки. Добавлено в API уровня 8 (Android 2.2.x). Возможными значениями являются `car` и `desk`.

- **Режим ночь** &ndash; ли приложение выполняется в ночное время или в день. Это может измениться в течение жизненного цикла приложения и предназначена для предоставления разработчикам возможности использовать более темные версии интерфейса ночью. Добавлено в API уровня 8 (Android 2.2.x). Возможными значениями являются `night` и `notnight`.

- **Экран разрешением (dpi)** &ndash; количество точек в данной области на физического экрана. Обычно выражается в виде точек на дюйм (dpi). Доступны следующие значения:

    - `ldpi` &ndash; Низкая плотность экранов.

    - `mdpi` &ndash; Средняя плотность экранов

    - `hdpi` &ndash; Высокая плотность экранов

    - `xhdpi` &ndash; Очень высокая плотность экранов

    - `nodpi` &ndash; Ресурсы, которые не должны быть масштабирован

    - `tvdpi` &ndash; Появился в уровень API 13 (Android 3.2) для экранов между mdpi и hdpi.

- **Тип сенсорного экрана** &ndash; указывает тип сенсорный экран на устройстве может быть. Возможными значениями являются `notouch` (не сенсорный экран) `stylus` (активной сенсорный подходит для пера) и `finger` (сенсорного экрана).

- **Сочетания доступности** &ndash; указывает, какие сочетания доступны. Это может измениться в течение жизненного цикла приложения &ndash; например в том случае, когда пользователь открывает аппаратную клавиатуру. Доступны следующие значения:

    - `keysexposed` &ndash; Устройство имеет клавиатура доступна. Если клавиатура не включена, затем это используется только при открытии аппаратную клавиатуру.

    - `keyshidden` &ndash; На устройстве нет аппаратной клавиатуре, но она скрыта и не сочетания программного обеспечения включена.

    - `keyssoft` &ndash; устройство имеет клавиатуру программного обеспечения включена.

- **Основной способ ввода текста** &ndash; используйте, чтобы указать, какие виды оборудования ключи доступны для ввода данных. Доступны следующие значения:

    - `nokeys` &ndash; Отсутствуют ключи оборудования для входных данных.

    - `qwerty` &ndash; Нет доступных клавиатуры qwerty.

    - `12key` &ndash; Отсутствует ключ 12 клавиатура


- **Доступность навигации ключ** &ndash; для когда доступна способом 5 или d панель навигации (письмом клавиатура). Это можно изменить во время существования приложения. Доступны следующие значения:

    - `navexposed` &ndash; клавиши перехода доступны пользователю

    - `navhidden` &ndash; клавиши перехода недоступны.

-  **Основной метод навигации не поддерживающем сенсорное управление** &ndash; вида навигации доступен на устройстве. Доступны следующие значения:

    - `nonav` &ndash; что средство просмотра только навигации является сенсорный экран

    - `dpad` &ndash; d панель (письмом клавиатура) доступно для навигации

    - `trackball` &ndash; устройство имеет трекболом для навигации

    - `wheel` &ndash; редко, где имеется одна или несколько письмом колеса, доступных

-  **Версия платформы (уровень API)** &ndash; уровень API, поддерживаемые устройством в формате v*N*, где *N* уровень API, который является целевым. Например, v11 будет целевой уровень API 11 (Android 3.0) устройства.


Более полные сведения о ресурсе квалификаторы. в разделе [предоставление ресурсов](http://developer.android.com/guide/topics/resources/providing-resources.html) на веб-сайте разработчиков решений Android.


## <a name="how-android-determines-what-resources-to-use"></a>Как Android определяет, какие ресурсы для использования

Это очень возможно и скорее всего, приложение будет содержать много ресурсов. Важно понимать, как Android будет выделить ресурсы для приложения при его выполнении на устройстве.

Android определяет базовый ресурсы путем итерации теста следующих правил:

- **Исключить противоречивое квалификаторы** &ndash; к примеру, если книжной ориентацией устройства, затем все каталоги альбомная ресурсов будут отклонены.

- **Игнорировать не поддерживается квалификаторы** &ndash; ко всем уровням API доступны не все квалификаторы. Если каталог ресурсов содержит квалификатор, для которого не поддерживается устройством, что каталог ресурсов игнорируются.

- **Определить следующий наибольший приоритет квалификатор** &ndash; приведенной выше таблице выберите следующий квалификатор наивысший приоритет (сверху вниз).

- **Сохранить все каталоги ресурсов для квалификатора** &ndash; при наличии любого ресурса каталогов, соответствующих квалификатор в приведенной выше таблице, выберите следующий квалификатор наивысший приоритет (сверху вниз).

Эти правила также показаны на следующей блок-схеме:

[![Блок-схема ресурсы](alternate-resources-images/flowchart-sml.png)](alternate-resources-images/flowchart.png#lightbox)

Когда система выполняет поиск ресурсов, связанных с плотность и не удается найти их, он предпримет для поиска других плотность определенных ресурсов и их масштабирования. Android не могут обязательно использовать ресурсы по умолчанию.
Например при поиске разъема ресурсов и он не доступен, высокой плотности версию ресурса можно выбрать Android ресурсов по умолчанию или средняя плотность. Это связано с высокой плотностью ресурсов может быть уменьшен с коэффициентом 0,5, что приведет к меньшее количество проблем видимость, чем масштабировании с уменьшением Средняя плотность ресурса, который потребуется коэффициент 0,75.

Например рассмотрим приложение, имеющее следующие каталоги drawable ресурсов.

    drawable
    drawable-en
    drawable-fr-rCA
    drawable-en-port
    drawable-en-notouch-12key
    drawable-en-port-ldpi
    drawable-port-ldpi
    drawable-port-notouch-12key

И запустите приложение на устройстве со следующей конфигурацией:

- **Языковой стандарт** &ndash; en-GB
- **Ориентация** &ndash; порт
- **На экране плотность** &ndash; hdpi
- **Тип сенсорного экрана** &ndash; notouch
- **Основной метод ввода** &ndash; 12key

Начинается с исключаются французские ресурсы, как они конфликтуют с языкового стандарта `en-GB`, оставляя нам:

    drawable
    drawable-en
    drawable-en-port
    drawable-en-notouch-12key
    drawable-en-port-ldpi
    drawable-port-ldpi
    drawable-port-notouch-12key

Затем первый квалификатор выбирается из приведенной выше таблице квалификаторы: MCC и кодом МНЧ. Нет ресурсов каталоги, содержащие этот квалификатор, поэтому код кодом MCC/МНЧ игнорируется.

Выбран следующий квалификатор, которой является языком. Есть ресурсы, которые соответствуют код языка. Все каталоги ресурсов, которые не соответствует коду языка `en` отклоняются, таким образом, в список ресурсов теперь:

    drawable-en-port
    drawable-en-notouch-12key
    drawable-en-port-ldpi

Далее квалификатор, который присутствует — для ориентации экрана, поэтому вся ресурсов каталоги, которые не соответствуют ориентацию экрана `port` исключаются:

    drawable-en-port
    drawable-en-port-ldpi

Далее следует квалификатор для плотности экрана `ldpi`, в результате исключения одного дополнительные каталог ресурса:

    drawable-en-port-ldpi

В результате этого процесса Android будет использовать drawable ресурсы в каталоге ресурсов `drawable-en-port-ldpi` для устройства.

> [!NOTE]
> Квалификаторы размер экрана предоставить одно исключение из этого процесса выбора. Это возможно для Android выбрать ресурсы, которые предназначены для экрана меньше, чем предоставляет текущее устройство. Например, большим экраном устройство может использовать ресурсы обеспечения стандартного по размеру экрана. Однако это не наоборот: то же самое устройство большом экране не будет использовать ресурсов, предоставленных для xlarge экрана. Если Android не удается найти набор ресурсов, соответствующий размер данного экрана, произойдет сбой в приложении.
