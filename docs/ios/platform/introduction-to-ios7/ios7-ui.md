# <a name="ios-7-user-interface-overview"></a>iOS 7 Обзор пользовательского интерфейса

_iOS 7 предоставляет множество изменения интерфейса пользователя. В этой статье кратко описаны больше изменений, внешний вид элементов управления и интерфейсы API, которые поддерживают новый дизайн._

iOS 7 фокусируется на содержимое через chrome. Элементы пользовательского интерфейса в iOS 7 Коррекция постискажений chrome, удалив атрибуты, такие как лишние границы строки состояния и панели навигации, которые уменьшают объем места на экране, используемые представления содержимого. В iOS 7 содержимое предназначено для использования во весь экран.

iOS 7 введено несколько изменений, другие: цвет используется для различения элементов пользовательского интерфейса, вместо атрибутов, таких как границы кнопки. Многие элементы, такие как панели навигации и строки состояния, стали размытой и полупрозрачными или прозрачными с содержимого представления область под ними ведения. Эти представления содержимого отрисовки через размытой столбцы, подразумевающий чувство глубины в пользовательском интерфейсе.

В этой статье рассматриваются некоторые изменения в элементы пользовательского интерфейса в iOS 7, а также различные интерфейсы API, связанные с для нового пользовательского интерфейса.

## <a name="view-and-control-changes"></a>Просмотр и управление изменениями

Все представления в UIKit соответствует нового внешнего вида iOS 7. В этом разделе описываются некоторые изменения в этих представлениях, а также связанные API, которые были изменены для поддержки нового пользовательского интерфейса.

### <a name="uibutton"></a>UIButton

Кнопки, созданные из `UIButton` класс теперь не имеет границ не фона по умолчанию, как показано ниже:

 ![](ios7-ui-images/button.png "Образец UIButton")

`UIButtonType.RoundedRect` Стиль рекомендуется к использованию. При использовании в iOS 7, `UIButtonType.RoundedRect` приведет к `UIButtonType.System` используется, что порождает стиль кнопки по умолчанию без фона или модели, как показано выше.

### <a name="uibarbuttonitem"></a>UIBarButtonItem

Аналогично `UIButton`, кнопки строки, также не имеет границ, по умолчанию принимается новый `UIBarButtonItemStyle.Plain` стиля, показано ниже:

 ![](ios7-ui-images/barbuttonplain.png "Образец UIBarButtonItem")

Кроме того `UIBarButtonItemStyle.Bordered` стиль рекомендуется к использованию. Установка `UIBarButtonItemStyle.Bordered` в iOS 7 приводит `UIBarButtonItemStyle.Plain` , используемый в стиле.

`UIBarButtonItemStyle.Done` Стиль не является устаревшим. Тем не менее он также создаст без рамки кнопку, только со стилем полужирным шрифтом, как показано:

 ![](ios7-ui-images/barbuttondone.png "Образец UIBarButtonItem в стиле Готово")

### <a name="uialertview"></a>UIAlertView

Помимо изменения стиля для новый внешний вид iOS 7 представления "предупреждения" больше не поддерживает настройку через вложенное представление. Несмотря на то что `UIAlertView` наследует от `UIView`, вызов `AddSubview` на `UIAlertView` не делает ничего. Рассмотрим следующий пример кода:

```csharp
UIBarButtonItem button = new UIBarButtonItem ("Bar Button", UIBarButtonItemStyle.Plain, (s,e) =>
{
    UIAlertView alert = new UIAlertView ("Title", "Message", null, "Cancel", "OK");

    alert.AddSubview (new UIView () {
        Frame = new CGRect(50, 50,100, 100),
        BackgroundColor = UIColor.Green
    });

    alert.Show ();
});
```

Стандартное представление предупреждений, создает с вложенное представление, которые обрабатываются, как показано ниже:

 ![](ios7-ui-images/alert.png "Образец UIAlertView")
 
 Примечание: UIAlertView устарела в iOS 8. Представление [предупреждения контроллера](https://developer.xamarin.com/recipes/ios/standard_controls/alertcontroller/) инструкций об использовании представления "предупреждения" в iOS 8 и более поздних версий.

### <a name="uisegmentedcontrol"></a>UISegmentedControl

Сегментированный элементов управления в iOS 7 являются прозрачными, а также поддерживает оттенок цвета. Цвет оттенка используется для цветов текста и границы. При выборе сегмент цвет меняется местами между фон и текст, с оттенок цвета, используемый для выделения выбранного сегмента, как показано ниже:

 ![](ios7-ui-images/segmentedcontrol.png "Образец UISegmentedControl")

Кроме того `UISegmentedControlStyle` был объявлен устаревшим в iOS 7.

### <a name="picker-views"></a>Выбор представления

API-Интерфейс для выбора представления практически без изменений; Однако правила 7 разработки iOS теперь состояния выбора представления должны быть представлены встроенные, а не как входной представления анимация от нижней части экрана или с помощью нового контроллера стек контроллер навигации, как в предыдущем iOS версии. Это показано в приложении календаря системы:

Панель поиска отображается в области навигации панели при `UISearchDisplayController.DisplaysSearchBarInNavigationBar` задано значение true. При значении false — значение по умолчанию — на панели навигации скрыта при отображении поиска контроллера.

API-интерфейсы вокруг `UITableView` являются главным образом без изменений; Однако стиль значительно изменены для обеспечения соответствия нового пользовательского интерфейса. Иерархия внутреннего представления также имеет несколько по-разному. Это изменение не влияет на большинство приложений, но это следует учитывать это.

## <a name="general-user-interface-changes"></a>Пользовательский интерфейс общие изменения

-  Содержимое на весь экран
-  Внешний вид панели

### <a name="full-screen-content"></a>Содержимое в полноэкранном режиме

iOS 7, предназначенное для приложений воспользоваться преимуществами во весь экран. Просмотр контроллеров появиться перекрывающихся в строке состояния и панель навигации — при наличии — в отличие от нижней части панели состояния и переходы.

При подготовке приложения для iOS 7, можно выровнять визуально с помощью представлений *интерфейс построителя* или *Xamarin iOS конструктор*. Также можно использовать один новые интерфейсы API для обработки содержимого в полноэкранном режиме, программными средствами. Эти API-интерфейсы, представленные ниже.

#### <a name="toplayoutguide-and-bottomlayoutguide"></a>TopLayoutGuide и BottomLayoutGuide

 `TopLayoutGuide` и `BottomLayoutGuide` использовать в качестве ссылки для которых представления следует начать или завершить, так, что содержимое не перекрывается полупрозрачные `UIKit` строки, как показано в следующем примере:

 [![](ios7-ui-images/clipped.png "Примеры содержимого, не накладываются полосой полупрозрачные UIKit")](ios7-ui-images/clipped.png#lightbox)

Эти API-интерфейсы могут использоваться для вычисления представления смещения от верхней или нижней части экрана и соответствующим образом настроить размещения содержимого:

```csharp
public override void ViewDidLayoutSubviews ()
{
    base.ViewDidLayoutSubviews ();

    if (UIDevice.CurrentDevice.CheckSystemVersion (7, 0)) { 
        nfloat displacement_y = this.TopLayoutGuide.Length;

        //load subviews with displacement
    }

}
```

Мы используем вычисленном выше, чтобы задать значение нашей `ImageView`его смещение от верхней части экрана, поэтому отображается полный образ:

 [![](ios7-ui-images/good2.png "Пример ImageViews смещения от верхней части экрана.")](ios7-ui-images/good2.png#lightbox)

Ссылаться на [ImageViewer](https://developer.xamarin.com/samples/mobile/iOS7-ui-updates) рабочий образец.

Значение смещения формируется динамически после добавления представления к иерархии, поэтому при попытке чтения `TopLayoutGuide` и `BottomLayoutGuide` значения в `ViewDidLoad` вернет 0. Вычислить значение представление загрузки — например, в `ViewDidLayoutSubviews`.

> [!IMPORTANT]
> `TopLayoutGuide` и `BottomLayoutGuide` являются устаревшими в iOS 11 им следует предпочесть новые зарезервированная область макета. Apple указано, что зарезервированная область с помощью совместим с iOS версии более ранней, чем iOS версии 11. Дополнительные сведения см. в разделе [обновление приложения для iOS 11](~/ios/platform/introduction-to-ios11/updating-your-app/visual-design.md#fullscreen) руководства.

#### <a name="edgesforextendedlayout"></a>EdgesForExtendedLayout

Этот API указывает, какие границы представления следует расширенных во весь экран, вне зависимости от панели Прозрачность. В iOS 7 панелей навигации и панели инструментов отображаются располагаются над контроллера представление — в отличие от в предыдущем iOS версий, где они не занимают то же пространство. Приложение iOS 7 фотографии показано значение по умолчанию `UIViewController.EdgesForExtendedLayout` значение `UIRectEdge.All`. Этот параметр заполняется всех четырех краев в представлении с содержимым, создание эффект перекрывающихся и полноэкранном режиме.

 [![](ios7-ui-images/photos.png "Образец EdgesForExtendedLayout")](ios7-ui-images/photos.png#lightbox)

Коснитесь изображения удаляет столбцы и приводятся в полноэкранном режиме изображения.

 [![](ios7-ui-images/photos2.png "EdgesForExtendedLayout по отношению к столбцам удален")](ios7-ui-images/photos2.png#lightbox)

Поскольку содержимое в полноэкранном режиме по умолчанию, приложений, настроенных для iOS 6 будет иметь части представления обрезается, как показано на снимке экрана ниже:

 [![](ios7-ui-images/clipped.png "Приложений, настроенных для iOS 6 будет иметь части представления обрезается, как показано на этом снимке экрана")](ios7-ui-images/clipped.png#lightbox)

Изменение `UIViewController.EdgesForExtendedLayout` настраивает свойство для данного поведения. Можно указать представления не заполнялся, краям, поэтому наши представление позволит избежать отображения содержимого в место, занимаемое навигации или панели инструментов (в каждом ориентация):

```csharp
if (UIDevice.CurrentDevice.CheckSystemVersion (7, 0)) { 
    this.EdgesForExtendedLayout = UIRectEdge.None;
}
```

В нашем приложении мы увидим представление снова перемещен, поэтому отображается полный образ:

 [![](ios7-ui-images/good.png "Пример с видимыми всего изображения")](ios7-ui-images/good.png#lightbox)

Обратите внимание, что при последствия `TopLayoutGuide/BottomLayoutGuide` и `EdgesForExtendedLayout` API-интерфейсы похожи, они предназначены для заполнения разных целей. Изменение `EdgesForExtendedLayout` параметр по умолчанию может решить усеченные представления в приложениях, предназначенных для iOS 6, но разработки хорошо iOS 7 должны соблюдать улучшения внешнего вида полноэкранном режиме, а также назначить полноэкранном режиме просмотра качества, полагаясь на `TopLayoutGuide` и `BottomLayoutGuide`правильно разместить содержимое, в которой должны обрабатываться в месте, удобном для пользователя.

Ссылаться на [ImageViewer](https://developer.xamarin.com/samples/mobile/iOS7-ui-updates) рабочий образец.

### <a name="status-and-navigation-bars"></a>Состояние и панели навигации

Строка состояния и панели навигации отображаются с прозрачностью. Строки состояния являются прозрачными, когда панели инструментов и строки навигации полупрозрачные и размытое для передачи чувство глубины в пользовательском интерфейсе. На следующем рисунке показан этот размытие и прозрачность, где показаны синего цвета фона представления коллекции — состояния и навигации панели, предоставляя им светлой синий внешний вид:

 ![](ios7-ui-images/transparent-navbar.png "Пример состояние и панель стирают навигации")

#### <a name="status-bar-styles"></a>Стили строки состояния

Вместе с размытие и прозрачности переднего плана в строке состояния может быть светлый или темный (темный используется по умолчанию). Можно задать стиль панели состояния из контроллера представления. Представление контроллер можно также задать ли показывается или скрывается в строке состояния.

Например, следующий код переопределяет `PreferredStatusBarStyle` метод контроллера представления вносить в строке состояния отображения светлой переднего плана:

```csharp
public override UIStatusBarStyle PreferredStatusBarStyle ()
{
    return UIStatusBarStyle.LightContent;
}
```

В результате строке состояния отображается как показано ниже:

 ![](ios7-ui-images/light-status-bar.png "Пример строки состояния")

Чтобы скрыть строку состояния из контроллера представление кода, переопределение `PrefersStatusBarHidden`, как показано ниже:

```csharp
public override bool PrefersStatusBarHidden ()
{
    return true;
}
```

Скрывает строку состояния:

 ![](ios7-ui-images/status-bar-hidden.png "Скрытые строки состояния")

### <a name="tint-color"></a>Оттенок цвета

Кнопки теперь отображаются как текст менее chrome. Цвет текста можно управлять с помощью нового `TintColor` свойство `UIView`. Параметр `TintColor` применение цвета всего представления иерархии для представления, устанавливает его. Чтобы применить `TintColor`по всему приложению, устанавливается для `Window`. Можно также определить при изменении цвета оттенка через `UIView.TintColorDidChange` метод.

Например, на следующем рисунке показан эффект от изменения оттенок цвета на контроллере навигации представления для фиолетовым:

 ![](ios7-ui-images/tint-color.png "Цвет фиолетовым цветом для представления контроллеров навигации")

Цвет оттенка может применяться для изображений, а также при `RenderingMode` равно `UIImageRenderingMode.AlwaysTemplate`.

> [!IMPORTANT]
> Цвет оттенка нельзя задать с помощью `UIAppearance`.


### <a name="dynamic-type"></a>Динамический тип

В iOS 7 пользователь может указать размер текста в параметрах системы. С помощью динамического типа шрифта корректируется динамически плохо, независимо от размера. `UIFont.PreferredFontForTextStyle` можно использовать для получения управляемых пользователем размер шрифта, который оптимизирован.
