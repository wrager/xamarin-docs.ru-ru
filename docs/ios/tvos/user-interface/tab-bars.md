# <a name="working-with-tab-bar-controller"></a>Работа с контроллером панель вкладок

_В этой статье описывается проектирование и работа с вкладки панели контроллера внутри приложения Xamarin.tvOS._

Для многих типов приложений tvOS переходов будет представлено в виде панели вкладок под управлением в верхней части экрана. Пользователь предъявляет влево и вправо по списку возможных категорий и область содержимого ниже изменения в соответствии с выбранным пользователем.

[![](tab-bars-images/tab01.png "Пример панели вкладки")](tab-bars-images/tab01.png#lightbox)

Панель вкладок прозрачная по умолчанию и всегда отображается в верхней части экрана. В фокусе, панель вкладок покрывают top 140 пикселей экрана, но будет быстро становятся видимыми, немедленно когда фокус перемещается к области содержимого.

<a name="Tab-Bars-in-tvOS" />

## <a name="tab-bars-in-tvos"></a>Вкладка панели в tvOS

`UITabViewController` Работает так же, как и служат одинаковой цели на tvOS, как и в iOS, со следующими отличиями ключа:

- В отличие от панели вкладок на iOS, которая отображается в нижней части экрана полосы вкладки в tvOS занимают top 140 пикселей экрана и полупрозрачных по умолчанию.
- При перемещении фокуса на вкладке панели область содержимого, на вкладке панели будет быстро слайд, off верхней части экрана и скрыт. Пользователь может кнопку меню или проведите вверх на [Siri удаленного](~/ios/tvos/platform/remote-bluetooth.md#The-Siri-Remote) Чтобы снова отобразить панель вкладок.
- Считывания вниз на удаленном Siri фокус ввода переместится в область содержимого ниже панель вкладок, чтобы первый [элемент может иметь фокус](~/ios/tvos/app-fundamentals/navigation-focus.md#Focus-and-Selection) отображаемого содержимого. Еще раз при этом будут скрыты панели вкладок после фокуса.
- Щелкните для выбора категорий, отображаемых на панели вкладок переключения, содержимое и основное категории переключается на первый элемент может иметь фокус в этом представлении.
- Число категорий, отображаемых на панели вкладок, должен быть решен и все категории должен быть доступен всегда, заданной категории никогда не должны быть отключены.
- Вкладка панели на tvOS не поддерживают настройки. Кроме того, они не отображают **дополнительные** категории (например, iOS), если существует несколько категорий, чем может поместиться на панели вкладок.

Apple имеет следующие рекомендации по работе с панелями вкладки:

- **Используйте вкладку панели, чтобы логически упорядочить содержимое** -позволяет логически упорядочить содержимое, которое приложение tvOS работает с панели вкладок. Например, основная, сверху диаграммы, Куплено и поиска.
- **Добавление эмблемы для информирования пользователей новое содержимое** -при необходимости можно вывести эмблему (красный Овал с белым числа или восклицательным знаком) для оповещения пользователя новое содержимое в категории.
- **Используйте эмблемы экономно** — не загромождать панель вкладок с эмблемы и вывести их где они предоставляют важную информацию для пользователя.
- **Ограничения на число категории** : чтобы снизить сложность и удержаться приложения управляемые, не перегружать вашей панели вкладки с категориями и убедитесь, что все категории доступны и не высока. Лучше всего простые, короткие заголовки.
- **Не отключить категории** -все вкладки (категории) всегда должны быть видимы и включены постоянно. Если данная вкладка не имеет содержимого, почему предоставить пояснение для пользователя. Например вкладка покупки будет пустым, если пользователь выбрал нет покупки.

<a name="Tab-Bar-Items" />

## <a name="tab-bar-items"></a>Вкладка панели элементов

Каждой категории (вкладка) в панели вкладок представлен элемент вкладки панели (`UITabBarItem`). Apple имеет следующие рекомендации по работе с вкладки панели элементов.

- **Используйте вкладки на основе текста** -пока элемент панели вкладки могут быть представлены в виде значка, Apple предлагает только потому, что краткий заголовок легче интерпретировать, чем значок, с помощью текста.
- **Используйте короткое, может применяться существительных или команд** -элемент панели вкладки должны четко ретрансляции содержимое, которое содержит и лучше всего работает при простое имя существительное (например, фотографии, видео или музыка) или команд (например, поиска или воспроизведения).

    [![](tab-bars-images/tab05.png "Перейдите на вкладку")](tab-bars-images/tab05.png#lightbox)

1. Предоставляют все необходимые события для работы с элементами управления пользовательского интерфейса в коде C#.
1. Имя любого элементов управления пользовательского интерфейса, которые требуется предоставить в коде C#.

## <a name="working-with-tab-bars"></a>Работа с полосы вкладки

Используйте `Items` свойство `UITabBar` доступ к коллекции из `UITabBarItems` он содержит как нуль (0) индексированного массива. `SelectedItem` Свойство будет возвращать выбранной вкладки (категория) как `UITabBarItem`.


<a name="Working-with-Tab-Bar-Items" />

## <a name="working-with-tab-bar-items"></a>Работа с элементами панели вкладки

Чтобы отобразить индикатор событий на данной вкладке (красный Овал с белым текстом), используйте следующий код:

```csharp
// Display a badge
TabBar.Items [2].BadgeValue = "10";
```

Бы их следующие результаты при выполнении:

[![](tab-bars-images/tab06.png "Элемент вкладки панели с эмблемы")](tab-bars-images/tab06.png#lightbox)

Используйте `Title` свойство `UITabBarItem` изменение заголовка и `Image` свойства, чтобы изменить значок.
